# ml4js v0.2 規劃

## 目標

在 `v0.1` 的基礎上，把使用流程推進到更像 scikit-learn：

```ts
const { xTrain, xTest, yTrain, yTest } = trainTestSplit(X, y, {
  testSize: 0.33,
  seed: 7
});

const pipeline = new Pipeline(
  [{ name: 'scale', transformer: new StandardScaler() }],
  new LinearRegression()
);

pipeline.fit(xTrain, yTrain);
const predictions = pipeline.predict(xTest);
const score = pipeline.score(xTest, yTest);
```

## 功能範圍

1. `trainTestSplit`
   - 支援 `testSize`
   - 支援 `shuffle`
   - 支援 `seed`
2. `Pipeline`
   - 支援多個前處理步驟
   - 支援最終回歸器 `fit/predict/score`
3. 測試補強
   - `trainTestSplit` 單元測試
   - `Pipeline` 單元測試
   - 系統流程改成使用 split + pipeline

## 設計原則

1. 先只支援回歸流程，不提前做分類器泛化。
2. `Pipeline` 保持最小化，不加入參數搜尋或複雜的 step 操作。
3. 隨機切分必須可重現，方便測試與教學。
